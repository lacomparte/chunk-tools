캐러셀(carousel) 구현하기

#설명

- 당신은 가상의 회사 ʻ춘식컴퍼니’ 에 재직 중입니다. 웹페이지 개발을 의뢰 받은 상황이라 가정하시고 아래 고객의 요구사항에 맞게 프로젝트를 수행해주세요!

# 고객의 요구사항

- “우리 회사의 뉴스를 소개할 수 있는 웹 페이지를 추가하고 싶습니다. 아래 조건에 맞게 만들어주세요!”
- 화면 상단에는 헤드라인 뉴스의 내용이 캐러셀(Carousel)\* 슬라이드로 보여지고, 화면 하단에는
  뉴스 목록이 보여지는 페이지를 개발해주세요. 아래에 디자인 시안 목업을 첨부했는데, 참고만
  해주세요.
- 첨부 파일로 제공한 “newsData.json” 내용을 사용해서 상단 캐러셀과 하단 뉴스목록을
  개발해주세요. (상단 캐러셀 데이터 : headLineNews, 하단 뉴스목록 데이터 : commonNews)
- 캐러셀(Carousel) : 왼쪽에는 이전 슬라이드가, 오른쪽에는 다음 슬라이드가 보여지며 무한하게
  슬라이딩이 가능합니다. 슬라이드 개수가 100개 넘을 수도 있으니, 슬라이드의 개수와 상관없이
  사용자가 사용하기 편한 방식으로 개발해주세요.
  그리고 캐러셀 아래의 페이지네이션 (또는 인디케이터 요소)을 통해 언제든 원하는 슬라이드로
  쉽게 이동할 수 있어야 합니다. 유형과 스타일은 자유입니다.
- 반응형 : 많은 사람들이 볼 수 있는 서비스인 만큼, 데스크탑과 모바일 환경에 맞게 반응형으로
  보여야 합니다.
- 개발환경 : 다음 페이지의 “개발환경”에 맞추어 개발해주세요.
- 사회적인 기업인 만큼 웹접근성도 고려하여 개발해주세요.

# 디자인 시안 목업

- 상단 <headLineNews> 컴포넌트의 carousel 각 아이템의 width 는 모두 일정하지 않다. width 와 height 는 mediaquery 가 적용되어야 한다.
- 하단 <commonNews> 는 viewport 의 크기에 따라 mediaquery 적용으로 디자인이 변경되어야 한다.
  - 디자인이 변경되는 break-point 는 인지학적 기준을 따른다.
  - 작은 단말기 일때는(ex. mobile 환경) image 가 100% 꽉차게 구현하고 제목/내용/날짜는 한 덩어리로 처리하고 각 라인별 개행처리 한다.
  - 작은 단말기가 아닐때는 (ex. ipad) 아이템의 좌우 패딩이 존재하고, flex 를 사용하여 2col 로 구현한다.

# 개발 환경

- Vite, React, Typescript 를 사용해주세요.
- 상태관리를 위한 라이브러리의 사용은 가능하지만, 로직에 영향을 주는 라이브러리는 사용하지
  않고 개발해주세요.
- 뉴스데이터는 JSON 으로 제공합니다.
- Vite 서버 포트는 3000으로 설정해주세요. (host: true, port: 3000)

# 추가 설명

- 캐러셀 (Carousel) : 캐러셀은 하나의 콘텐츠 영역 내에 여러 개의 서로 다른 콘텐츠를 표시할 수
  있는 컴포넌트이다. 캐러셀에 포함된 콘텐츠는 가로로 배치되어 있으며, 사용자가 좌/우로
  콘텐츠를 회전시켜 콘텐츠를 탐색하도록 하거나 자동으로 회전하도록 설정할 수 있다.
